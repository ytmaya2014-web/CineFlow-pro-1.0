📦
三种工作容器
›
临时对话
用完即弃,适合一次性问题。
PROJECT 项目
长期、多轮、共享背景。素材 + 方法论常驻。
SKILL 技能
可复用方法论,SKILL.md 打包,合适时自动调用。
⚖️
Chat vs Cowork
›
CHAT 思考伙伴
和它对谈,把想法逼出来。看不到你电脑文件,产出在对话里。
COWORK 执行同事
给它目标,它规划+执行,直接产出文件到本地。
判断标准:"帮我想" → Chat;"帮我做" → Cowork。

›
OPUS · 最强

Web Search 用法
›
问需要最新信息的问题它会自己判断,也可以说"搜一下..."强制触发。中文小众内容覆盖一般,重要资料还是自己上传 Knowledge。

长期管理(防越用越乱)
›
• 重要对话手动改名
• 单次对话超几十轮就拆分
• 每周更新 Knowledge
• 跨工具配合:成稿存本地,Knowledge 只放工作副本
桌面端 Cowork 流程
辅助战场:整理、归档、批量执行。
🎬
Cowork 的真正价值
›
让 Claude 不再只是对话伙伴,而是能直接整理素材、生成文档、跨工具串联工作流的"协作同事"。
心态切换:你是项目经理,不是聊天对象。

文件夹搭建原则
›
1. 单独建 Cowork 工作区,不要给根目录权限
2. 重要原稿放在它访问不到的地方
3. 第一次先用测试沙盒文件夹试手
4. 子目录按"01_原始 / 02_当前 / 03_参考 / 04_产出 / 05_归档"分

Dispatch · 手机派任务
›
手机扫码配对 Cowork 桌面会话。可以从手机派任务,回家看完成的产出。早期版本约 50% 成功率。
桌面端必须保持唤醒 + 应用打开,任务才能跑。
⚠️
必须知道的 5 个限制
›
1. 桌面应用必须打开 — 合上盖子任务就死
2. 用量消耗远高于 Chat — 长任务前心里有数
3. 研究预览阶段 — 复杂任务约 50% 成功率
4. 文件操作不可撤销 — 没有 Ctrl+Z
5. UI 在快速变动 — 路径不一致查官方文档

版本一 · 航向锁定
首次
›
项目首次会话,让 Claude 自我对齐 + 提议焦点。
Chat · 航向锁定
本轮对话定向 我刚进入这个项目(或新一轮对谈)。在我们开始之前,请用三句话回答: 1. 你从 Project Instructions 和 Project Knowledge 里识别到的核心方法论是什么? 2. 你认为这个项目当前的核心张力 / 待解问题是什么? 3. 基于以上,你建议本次对话聚焦在哪一件事? 回答完不要主动展开。等我决定本次对话的真正焦点。
🔬
版本二 · 诊断式深度对谈
分析
›
把"出报告"转化为"对谈节奏"。
Chat · 诊断启动
诊断启动 请按本项目的"双阶段诊断框架"分析我接下来贴的稿子。流程要求: 1. **不要写一份完整报告甩给我**。我们用对谈方式推进。 2. 先做骨骼层第一维度(内核锚定),给出你的判断。 3. 等我回应后再进入第二维度(结构拆解)。 4. 每个维度结束时,主动问我一个"逼我深想"的问题——而不是问"是否继续"。 5. 大锤清单留到最后,基于前面所有对谈的共识来列。 记住:你是诊断医生,不是写报告的实习生。 [在此贴稿子内容]
⚔️
版本三 · 议题深挖
硬核
›
逼出思考深度,拒绝方案列表。
Chat · 死磕这一件事
死磕这一件事 议题:[在此填具体问题] 我不要方案列表。我要你和我一起把这件事想穿。 请按以下节奏进行: 第一轮:你给出对这个问题的初步判断(不超过 5 行)。 第二轮:我反驳或追问后,你不要妥协,先坚持你的判断,补充证据。 第三轮:我们才进入"如果你说的对,会推出什么?"的推演。 第四轮:在推演基础上,我们一起评估 2-3 个具体路径。 绝对禁止: - 一上来就给我 5 个方案的对比表 - 用"这取决于你的偏好"这种回避性回答 - 没有立场地中立陈述

版本四 · 续接式
回归
›
久违回归,先重建上下文 + 重新协商焦点。
Chat · 对话续接
对话续接 我们之前在 [对话名称 / 时间] 讨论过 [话题]。 请基于 Project Knowledge 里的相关产出文件(或我即将贴出的总结),完成以下: 1. 用三句话复述上次我们达成的核心共识。 2. 列出上次留下的"未解决议题"。 3. 提议本轮我们应该优先处理哪一个,并说明为什么。 完成后等我确认本次焦点。不要直接开始处理。
🎁

必备

请总结本次对话: 1. 核心共识(不超过 3 条) 2. 未解决议题 3. 下一轮建议焦点 简短即可。我会把这份总结存到项目笔记里。

首次
›
首次会话,让 Cowork 自检 + 摸底 + 建议 + 等待指令。
Cowork · 启动协议
启动协议·首次会话 请按以下顺序执行,每完成一步停下来等我说"继续": 第一步:自检 - 复述你当前装载的全局指令核心要点(不超过 5 条) - 复述本项目文件夹专属指令核心要点(不超过 5 条) - 确认你已识别 Skill 是否可用 - 报告你当前可访问的工作文件夹路径 第二步:项目摸底 - 扫描所有子文件夹的文件清单 - 识别哪些是稿件、素材、参考资料、历史产出 - 找出最新版本(按修改时间) - 报告项目当前的"文件健康度" 第三步:工作建议 基于摸底结果,给我三个建议: 1. 当前最值得优先做的一件事(以及为什么) 2. 一件可以立刻做的"低成本快速验证"任务 3. 一件你建议我**暂时不要做**的事 第四步:等待指令 完成上述三步后,停下,不要主动开始任何文件操作。 等我说"开始 [具体任务]"再动手。 现在开始第一步。

Cowork · 项目复位
项目复位 我已经一段时间没动这个项目了。 请帮我做以下事情,不要执行任何写入操作: 1. 扫描 04_Claude产出/ 下所有文件,按时间倒序列出 2. 找出最近一份诊断报告,简要总结其核心结论(不超过 10 行) 3. 对比这份诊断报告和当前最新稿: - 诊断里指出的问题,新稿子有没有处理?哪些处理了,哪些没处理? - 是否需要新一轮诊断,还是可以推进到下一阶段? 4. 给我三个选项,标注每个的预估工作量(轻/中/重): 选项 A:某件事 选项 B:某件事 选项 C:某件事 完成后等我选,不要自动开始。

# 用户身份与工作模式 我是 [职业身份],使用 Cowork 处理素材整理、稿件诊断、文档生成、跨工具研究。 Chat 是想问题,Cowork 是派活——你执行,我审阅。 # 沟通风格 1. 直接,不啰嗦。不要"当然!""我很乐意帮您"开场白。 2. 不要每段后加情绪安抚。 3. 重要判断必须给理由,但理由要短。 4. 不确定的事明确说"不确定"。 5. 中文输出。 # 文件操作硬规则(不可违反) 1. 任何"删除""覆盖""归档"操作,执行前必须先列出影响的文件清单并等待我确认。 2. 修改任何已存在的文件之前,先创建一份带时间戳的备份副本。 3. 不允许在指定工作文件夹之外创建、读取、修改任何文件。 4. 生成新文件命名格式:用途_日期_版本号(如 诊断报告_20260416_v1.md)。 5. 默认产出位置:工作文件夹下的 04_Claude产出/ 子目录。 # 任务执行工作流 每接到一个任务,按以下顺序响应: 第一步:任务理解 - 用一句话复述我要的结果 - 列出关键约束 - 如有歧义,只问一次最关键的那个问题 第二步:执行规划 - 列出计划执行的步骤 - 标注哪些步骤涉及不可逆操作 第三步:征求许可 - 等我明确说"开始""执行""go"才动手 第四步:执行与汇报 - 每完成一个主要步骤简短汇报(一行内) - 遇到意外立刻停下问我 - 完成后给"完成总结" # 不要做的事 1. 不要自作主张优化我的文件结构、重命名我的文件 2. 不要在产出里加"免责声明"等兜底文字 3. 不要滥用 emoji 和大量加粗 4. 不要把"完成总结"写成长篇报告 5. 不要在我没问的时候提建议 # 用量意识 - 不要做不必要的"二次确认型"读取 - 不要为"严谨"把所有相关文件全扫 - 任务结束就结束,不要主动开启新一轮

避坑清单
8 个最常见的错误用法,直接对照纠正。
✗在 Chat 里挤牙膏式给信息
✓一次性贴齐所有上下文
✗一个对话里塞 5 个不相关问题
✓一轮聚焦一件事
✗Chat 对话从不收口
✓每次结束都强制收口,把总结存档
✗让 Cowork 访问整个硬盘
✓只授权专门的工作文件夹
✗Cowork 任务说一句模糊指令
✓五要素齐全 + 先要计划
✗Cowork 产出直接当最终稿
✓必须人工打开复核
✗把 Chat 和 Cowork 当同一种工具用
✓思考用 Chat,执行用 Cowork
✗全程用同一个模型
✓深度任务 Opus,迭代任务 Sonnet
