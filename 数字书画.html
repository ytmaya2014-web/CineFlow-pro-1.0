系统架构
第一层 · 分布式数据存储
PostgreSQL 存元数据 + IPFS 存图像/3D文件，哈希值双向关联
Node.js · PostgreSQL · IPFS · Python
第二层 · 区块链确权
ERC-721 合约铸造 NFT，链上锚定 IPFS 哈希，永久不可篡改
Solidity · Hardhat · Web3.js · EVM链
第三层 · P2P 交易市场
去中心化撮合，书画币(SHC)作为唯一流通媒介，多签钱包托管
Solidity(DEX) · React · ethers.js · MetaMask
第四层 · 全球共识治理 (规划中)
DAO 投票，博物馆/鉴定机构多方参与，质押 SHC 获治理权
规划阶段
数据流向
上传 扫描/拍摄文件 → IPFS 节点存储 → 生成唯一哈希
录入 元数据结构化写入 PostgreSQL 数据库
上链 调用智能合约，IPFS 哈希写入区块链，铸造 NFT
交易 NFT 在 DEX 挂单，买方用 SHC 支付，合约自动结算
给开发团队
· 此为前端原型演示，数据均为模拟
· 区块链交互需接入 MetaMask 钱包
· IPFS 建议使用 Pinata 或自建节点
· 数据库推荐 PostgreSQL 15+
· 书画币参考 ERC-20，总量 2100 万枚
· 建议优先完成第一阶段再接入链上
