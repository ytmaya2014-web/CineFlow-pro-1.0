🧠 模型家族

Claude 模型家族介绍

Anthropic 旗下有多个不同定位的模型，选对模型能让工作效率翻倍。

四大模型对比
Claude Opus
旗舰级 · 最强推理
最强大脑
Anthropic 最顶尖的模型，具备极强的复杂推理、深度分析和创意生成能力。处理长文档、复杂策略、多步骤任务时表现卓越。
适合：复杂创作（如心域三部曲这类深度项目）、长文档综合分析、需要深度思考的策划任务
Claude Sonnet
主力型 · 性能与速度均衡
日常主力
性能与速度的最佳平衡点，是绝大多数用户的首选。能处理复杂任务，速度比 Opus 快，成本比 Opus 低很多。Claude Code 终端默认使用 Sonnet。
适合：日常工作流、文件分析、代码编写、终端任务执行，是你最常用的模型
Claude Haiku
轻量级 · 极速响应
快速轻量
最快、最便宜的模型，适合简单任务和高频率调用。响应速度极快，API 成本极低，适合需要大量调用的场景。
适合：简单问答、格式整理、内容摘要、高频 API 调用、不需要复杂推理的重复任务
Claude Code
终端工具 · 代码专家
开发利器
专门为代码和文件操作设计的终端工具（不是模型名称，而是一个工具）。可以直接读写本地文件，在项目文件夹里自动执行任务，像 Cursor 一样工作。
适合：本地文件处理、自动写入 md 文件、执行项目工作流，就是本手册的核心工具

网页端 Projects 完整操作手册

专为第一次使用 Claude 的小白准备。把文件变成可对话的知识库，上传一次，永久记住，像聊天一样提问。

先理解：普通对话 vs Projects
普通对话Projects（项目）
文件记忆每次聊天结束就忘了✅ 上传一次，永久记住
背景记忆没有，每次从头开始✅ 进项目就记得上次的背景
适合随手问问题✅ 长期项目、多个文件
💡Projects 就是一个有"记忆"的专属工作空间，你把文件放进去，它就一直记得，不用每次重新上传。

第二部分：上传你的文件
4

在项目设置里找到 "Project Instructions" 或 "项目说明" 输入框。

这是什么？就是告诉 Claude：你是谁、这个项目是干什么的、你希望它怎么工作。每次你在这个项目里发消息，它都会先读这段说明，再回答你。

如何生成一个可以保存的 MD 文件
💡网页端 Claude 本身无法直接在你电脑上创建文件，但你可以让它生成完整的 md 文件内容，然后自己手动保存。
1
让 Claude 生成完整 md 内容
在对话框输入下方模板，Claude 会输出一大段 markdown 格式的文字。
2
复制全部内容
找到 Claude 回答右下角的复制按钮（两个方块叠在一起的图标），点一下，全部内容就复制到剪贴板了。
3
安装 VS Code（只需装一次）
下载地址：code.visualstudio.com
Mac：下载后把图标拖入「应用程序」文件夹即完成安装。
也可用 Mac 自带 TextEdit（备忘录也可以）。
4
新建文件，粘贴，保存为 .md
VS Code：Cmd+N 新建 → 粘贴内容 → Cmd+S 保存
文件名写：项目名_执行链.md，保存到项目文件夹

Mac TextEdit：格式 → 转换为纯文本 → 保存，文件名改为 项目名_执行链.md
5
把文件放进项目文件夹
把 .md 文件移动到你的本地项目文件夹里，和其他资料放在一起。之后用 Claude Code 终端时，它就能直接读取这个文件，按步骤执行并把答案写进去。
⚠️文件名必须以 .md 结尾，不能是 .txt，否则 Claude Code 终端读取时格式会乱。
生成执行链的提问模板
请根据项目里所有文件，帮我生成一份完整的执行链 md 文件。
要求：
- 用标准 markdown 格式
- 包含 Step 0 到 Step N 的完整结构
- 每步列明：输入文件、执行步骤、产出内容
- 产出位置留空，等我后续填入
第五部分：把答案保存到本地文件
9
复制答案到本地 md 文件
这是网页端和 Claude Code 终端最大的区别：
• Claude Code 终端：自动把答案写进你电脑的文件
• 网页端 Projects：需要你手动复制粘贴

操作：Claude 回答完 → 点复制按钮 → 打开执行链 .md 文件 → 找到对应步骤位置 → 粘贴 → 保存
让输出更好复制的提示词
请把以上内容重新整理，用标准 markdown 格式输出，
包括标题、列表、加粗，方便我直接粘贴到 md 文件里。

Claude 使用规范（来自用户配置）

1. 默认使用计划模式
   - 对于任何非简单任务（3个以上步骤或涉及架构决策），进入计划模式
   - 如果出现偏差，立即停止并重新规划——不要继续硬推
   - 验证步骤也要使用计划模式，不只是构建时
   - 提前写好详细规范以减少歧义

2. 子代理策略
   - 自由使用子代理以保持主上下文窗口整洁
   - 将研究、探索和并行分析任务交给子代理
   - 对于复杂问题，通过子代理投入更多算力
   - 每个子代理只负责一个任务，以保持专注

3. 自我改进循环
   - 用户做出任何纠正后：将该模式更新到 tasks/lessons.md
   - 为自己编写规则以防止同类错误再次发生
   - 不断迭代这些经验教训，直到错误率下降
   - 在每次会话开始时回顾相关项目的经验教训

4. 完成前验证
   - 不证明有效就不要标记任务为完成
   - 问自己："一位高级工程师会批准这个吗？"
   - 运行测试、检查日志、证明正确性

5. 追求优雅（但要平衡）
   - 对于非简单的更改：暂停并问"有没有更优雅的方式？"
   - 对于简单、明显的修复则跳过此步骤——不要过度工程化

6. 自主修复Bug
   - 收到bug报告时：直接修复，不要等用户手把手指导
   - 定位日志、错误、失败的测试——然后解决它们

任务管理
1. 计划优先：将计划写入 tasks/todo.md，带有可勾选的项目
2. 验证计划：开始实施前先确认
3. 跟踪进度：边做边标记完成项
4. 解释变更：每一步都做高层级总结
5. 记录结果：在 tasks/todo.md 中添加回顾部分
6. 积累经验：纠正后更新 tasks/lessons.md

核心原则
- 简单优先：让每个改动尽可能简单。代码影响最小化。
- 不许偷懒：找到根本原因。不做临时修复。保持高级开发者标准。
- 最小影响：改动只触及必要部分。避免引入新bug。

第五部分：常见问题
Claude Code 需要 Anthropic Key，我只有 Gemini Key 怎么办？▼
用 Gemini CLI 代替，效果类似：

文件太多，Claude 读得慢怎么办？▼
在提问时指定具体文件，不要让它扫描全部：「请只读取以下3个文件，回答我的问题：文件A、文件B、文件C」
每次重新打开终端都要重新 cd 进文件夹吗？▼
是的，但有快捷方式：在 Finder 里右键项目文件夹，选"新建位于文件夹位置的终端窗口"（需要在系统设置里开启此功能）。也可以把文件夹图标直接拖进终端，路径自动填入。
执行链.md 里内容越来越多，Claude 会不会搞混？▼
不会，Claude Code 每次都重新读取文件。但如果文件太大，可以把完成的步骤移到归档文件里，只保留当前进行的步骤在主文件里。
可以同时开多个项目吗？▼
可以，每个项目开一个终端窗口，分别 cd 进各自的文件夹，互不干扰。

编程入门

编程核心概念

搞懂这 5 个概念，后面学什么都快。

变量
变量 — 存东西的盒子
贴了标签的盒子，把数据放进去随时取用。
name = "张三"　把"张三"存在叫 name 的盒子里
函数
函数 — 可复用的流程
把经常用的操作打包起来，起个名字，下次直接调用。就像「发送邮件」这个动作，不用每次重新解释怎么发。
循环
循环 — 重复做同一件事
「对列表里每个人发一封邮件」——不用写 100 次，计算机替你重复。
条件
条件 — 判断做不同的事
「如果下雨带伞，否则不带」——程序根据不同情况走不同的路（if/else）。
API
API — 和外部服务沟通的接口
就像餐厅服务员：你说要什么（请求），他去厨房取来（响应）。Claude API 就是这个原理：你发消息，它返回回答。

重要提醒

小白注意事项

这些坑很多人踩过，提前知道就能避免。

提问技巧
✓
描述越具体，回答越准确
❌「帮我写个方案」
✅「帮我写针对25-35岁女性的护肤品推广方案，主打天然成分，预算5万，渠道小红书+抖音」
✓
告诉它输出格式
「请用 markdown 格式」「请用表格整理」「请分点列出」「请控制在 200 字以内」
✓
不满意直接说
「太长了，缩短到 100 字」「风格太正式，改口语化」「第 3 点展开详细说」
✓
给它设定角色
「你是资深纪录片导演，帮我…」— 角色设定能显著提升回答质量
常见错误（避坑）
!
不要完全相信输出
Claude 可能会编造看起来合理但不存在的信息（数据、引用等）。重要信息务必核实。

常见问题 FAQ

点击问题展开答案。

每次进 Projects 都要重新上传文件吗？▼
不需要。文件上传一次后永久保留在这个项目里。除非文件有更新，才需要重新上传新版本。
Project Instructions 每次都要重新写吗？▼
不需要。写一次后永久保存，每次进这个项目自动生效。
只有 Gemini Key，可以用终端吗？▼
可以，用 Gemini CLI：

npm install -g @google/gemini-cli

进入项目文件夹后输入 gemini 启动，使用方式和 Claude Code 类似。
XMind 文件怎么让 Claude 读取？▼
Claude 不支持直接读取 .xmind 格式。在 XMind 里：文件 → 导出 → PDF → 保存，然后上传或放入项目文件夹。
扫描版 PDF 读不出来怎么办？▼
扫描版 PDF 是图片，Claude 无法提取文字。用 Adobe Acrobat 或手机扫描 App（如扫描全能王）做 OCR 识别后重新导出。
每次重开终端都要 cd 进文件夹吗？▼
是的，但有快捷方式：在 Finder 里把项目文件夹图标直接拖进终端窗口，路径自动填入。
iOS 开发一定要 Mac 吗？▼
是的，Xcode 只能在 Mac 上运行。Windows 无法原生开发 iOS App。没有 Mac 可以考虑用 Flutter 做跨平台 App。
文件太多，Claude 回答不准怎么办？▼
在提问时指定具体文件：「请只读取以下 3 个文件回答我的问题：文件A、文件B、文件C」，不要让它扫描全部。
Opus / Sonnet / Haiku 有什么区别，我该用哪个？▼
Opus = 最强，适合复杂深度任务；Sonnet = 日常主力，性价比最高；Haiku = 最快最省，适合简单任务。日常用 Sonnet 就够了，重要的深度创作项目可以换 Opus。详见「模型家族介绍」章节。
网页端对话记录会保存吗？▼
会，在项目里的对话历史会保留。但每次新开一个对话，Claude 不会自动记得上次聊了什么，需要你把重要结论写进 Project Instructions 或重新告知。
