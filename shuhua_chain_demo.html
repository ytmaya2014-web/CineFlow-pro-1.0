系统架构说明
第一层：分布式数据存储
PostgreSQL（元数据）+ IPFS（图像/3D文件）双层存储，哈希值关联
技术栈：Node.js · PostgreSQL · IPFS · Python(图像处理)
第二层：区块链确权
ERC-721 智能合约铸造 NFT，链上存储 IPFS 哈希，实现不可篡改确权
技术栈：Solidity · Hardhat · Web3.js · Ethereum兼容链
第三层：P2P 交易市场
去中心化撮合引擎，书画币(SHC)作为唯一交易媒介，多签钱包安全托管
技术栈：Solidity(DEX合约) · React · ethers.js · MetaMask
第四层：全球共识治理
DAO 投票机制，博物馆/鉴定机构多方参与，质押 SHC 获得治理权
规划中
数据流向说明
上传 → 扫描/拍摄 → 图像文件上传至 IPFS
录入 → 元数据结构化写入 PostgreSQL 数据库
上链 → 调用智能合约，IPFS哈希写入区块链
交易 → NFT 在 DEX 挂单，买方用 SHC 支付
结算 → 智能合约自动转移 NFT + SHC，链上结算
给开发团队的备注
· 此演示为前端原型，所有数据为模拟数据
· 区块链交互需接入 Web3 钱包（MetaMask）
· IPFS 节点建议使用 Pinata 或自建节点
· 数据库建议 PostgreSQL 15 + TimescaleDB
· 书画币合约参考 ERC-20，总量 2100万枚
· 建议优先实现第一阶段，再逐步接入链上功能
